# 数据库设计规范

MySQL 库表设计、字段定义、索引策略及 SQL 编写规范。

## 适用范围

| 项目 | 说明 |
|------|------|
| 适用 | OLTP 业务库（用户、订单等） |
| 不适用 | OLAP 数仓、第三方系统库 |
| 命名风格 | **snake_case** 小写下划线 |

## 命名规范

### 库命名

- 格式：`业务前缀_场景`，如 `user_center`、`order_service`
- ❌ 不包含环境信息（`_dev`、`_test`）

### 表命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 实体表 | 名词复数/集合含义 | `user`、`order_item` |
| 关联表 | `主表_从表` | `user_role` |
| 临时表 | `tmp_` 前缀 | `tmp_xxx` |
| 备份表 | `bak_表名_YYYYMMDD` | `bak_order_20250101` |

❌ 禁止 `t1`、`tmp`、`abc` 等无意义命名

## 字段规范

### 命名

- snake_case，如 `user_id`、`created_time`
- 外键：`关联实体_id`，如 `product_id`
- 状态字段统一用 `status`

### 数据类型

| 类型 | 规范 |
|------|------|
| 主键 | `BIGINT`（雪花ID），避免自增 |
| 字符串 | `VARCHAR`，合理评估长度，❌ 滥用 `VARCHAR(255)` |
| 时间 | `DATETIME`/`TIMESTAMP` |
| 金额 | `DECIMAL`，❌ `FLOAT`/`DOUBLE` |
| 布尔 | `TINYINT`，默认 `0` |

### 审计字段

| 字段 | 说明 |
|------|------|
| `created_time` | 创建时间 |
| `updated_time` | 更新时间 |
| `created_by` | 创建人（可选） |
| `updated_by` | 更新人（可选） |

### 软删除

- 字段：`is_deleted TINYINT(1)`
- `0`=未删除，`1`=已删除
- 查询默认加 `is_deleted = 0`

## 索引规范

### 命名

| 类型 | 格式 | 示例 |
|------|------|------|
| 普通索引 | `idx_表名_字段` | `idx_order_user_id_created_time` |
| 唯一索引 | `uk_表名_字段` | `uk_user_login_name` |

### 设计原则

- 为高频查询、排序、关联字段建索引
- 组合索引：高区分度 + 常用过滤条件在前
- 单表索引数控制在 **3-5个**

### ❌ 禁止破坏索引

```sql
-- ❌ 错误：函数运算
WHERE DATE(create_time) = '2025-09-22'

-- ✅ 正确：范围查询
WHERE create_time >= #{startTime} AND create_time < #{endTime}
```

## 约束规范

| 约束 | 规范 |
|------|------|
| 唯一约束 | 全局唯一字段用唯一索引 |
| 外键约束 | ❌ **不建议物理外键**，应用层保证一致性 |
| 枚举约束 | 状态字段配套字典/枚举，避免魔法数字 |

## SQL 编写规范

### 禁止项

| 规范 | 说明 |
|------|------|
| ❌ 避免 JOIN | 高并发场景避免连表，应用层组装 |
| ❌ 避免复杂子查询 | 拆为多次简单查询 |
| ❌ 禁止破坏索引 | WHERE 中不对索引列用函数/计算 |
| ❌ 严禁 CASE WHEN | 业务逻辑下沉到服务层 |
| ❌ 禁止循环 SQL | 用批量或 IN 查询 |
| ❌ 严禁 SELECT * | 显式列出字段 |

### 分页优化

- 设计合理排序字段和索引
- 避免 `LIMIT 100000, 20` 大偏移量

## 事务与并发

| 规范 | 说明 |
|------|------|
| 事务隔离 | 默认级别，结合分布式锁实现业务隔离 |
| 控制粒度 | ❌ 事务中禁止远程调用/耗时IO |
| 死锁规避 | 多表更新顺序保持一致 |
| 关键写入 | 先获取分布式锁，再操作数据库 |
| 幂等控制 | 唯一索引防重复写入 |

## 安全规范

| 规范 | 要求 |
|------|------|
| SQL注入防护 | ❌ 禁止字符串拼接，必须用参数化查询 |
| 敏感数据 | 加密或脱敏存储 |
| 权限最小化 | ❌ 禁止 `DROP`/`TRUNCATE`，按业务域隔离账号 |
