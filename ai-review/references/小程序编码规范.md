# 小程序编码规范

本规范适用于 Taro 3 + React 18 + TypeScript 小程序项目，目标是：在保证良好用户体验的前提下，避免白屏、崩溃、严重性能问题和安全漏洞。

**加粗条目**表示 AI 代码评审的必检项。

---

## 目录与命名规范

### 工程目录结构

```
src/
  pages/             # 主包页面（tab 页等核心页面）
  subpackages/       # 按业务域划分的分包页面
  dynamicComponents/ # 动态加载的组件（必须异步引入）
  fq-common/         # 跨项目公共库（禁止直接修改）
  helpers/           # 工具类 helper（hXxx 命名）
  hooks/             # 通用 React Hooks
  consts/            # 常量与枚举
  utils/             # 纯函数工具
  scss/              # 全局样式
```

- 避免目录嵌套超过 4 级
- **不得在业务代码中直接修改 `fq-common/`**

### 文件命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 源码文件 | kebab-case | `user-profile.tsx` |
| 组件文件 | 反映职责 | `OrderList/index.tsx` |
| 样式文件 | 与组件同名 | `index.scss` |
| 类名 | BEM 或语义化 | 避免无意义缩写 |

### 文件头注释（ESLint 强制）

```typescript
/**
 * @Owners yourName
 * @Title 文件职责简述
 */
```

## 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 变量/函数 | lowerCamelCase | `userName`、`loadUserList()` |
| 常量/枚举 | 全大写下划线 | `MAX_RETRY_COUNT` |
| Storage Key | 定义在 `cStorageKeys.ts` | **禁止硬编码字符串 key** |
| 组件 | PascalCase | `UserProfile` |
| 对外事件 props | onXxx | `onSubmit`、`onChange` |
| 内部事件处理 | handleXxx | `handleClick` |
| Helper 工具类 | hXxx | `hRequest`、`hLogger` |

**禁止在业务代码中散落魔法数字或魔法字符串**

## 组件设计与状态管理

### 组件拆分

- **单文件不得超过 500 行**（不含类型定义和注释）
- 高频更新区域拆分为独立组件，避免父组件重渲染
- 合理使用 `React.memo`、`useMemo`、`useCallback`

### 动态组件引入（ESLint 强制）

`dynamicComponents/` 目录下的组件，**在外部必须使用异步 `import()` 引入**。

```typescript
// ✅ 正确
const CartModal = Taro.lazy(() => import('dynamicComponents/cart/CartModal'));

// ❌ 错误：同步引入（ESLint 报错）
import CartModal from 'dynamicComponents/cart/CartModal';
```

### 页面间代码隔离（ESLint 强制）

**禁止在一个页面中 import 另一个页面的入口文件**。共享逻辑提取到 `helpers/`、`hooks/` 或 `utils/`。

### 状态设计

- 只将 UI 渲染必需的数据放入 `useState` 或全局 store
- 避免深层嵌套对象直接作为状态，优先用展开运算符做不可变更新
- 全局 store 只存放跨页面共享的核心数据

### 列表渲染

- **禁止使用数组索引作为 `key`**（纯静态骨架屏除外）
- 大型列表优先考虑虚拟列表、分页或懒加载

| 组件 | 用途 |
|------|------|

## 图片资源规范（ESLint 强制）

**所有网络图片必须经过 `hOssAction.resizeImageToWebpAndInterlace()` 处理**。

```typescript
// ✅ 正确
<Image src={hOssAction.resizeImageToWebpAndInterlace(item.imageUrl)} />

// ❌ 错误（ESLint 警告）
<Image src={item.imageUrl} />
```

本地静态资源（Sprite 图、tabbar 图标）除外。

---

## 网络请求与错误处理

- **所有 API 请求必须使用 `hRequest` 封装**，禁止直接调用 `Taro.request()`
- **禁止空 `catch` 块**（`catch (e) {}`）和无回调的 `.catch()`
- catch 块内至少调用 `hLogger.error()` 或向用户展示错误提示
- 用户界面需体现失败状态，避免长时间白屏或 Loading

```typescript
// ✅ 正确
try {
    const res = await hRequest.api.goods.detail({ id });
    setDetail(res);
} catch (error) {
    hLogger.error({ desc: 'goods detail fetch error', error });
    hTaro.showToast('加载失败，请重试');
}

// ❌ 错误：空 catch
try { ... } catch (e) {}

// ❌ 错误：.catch() 无回调
fetchData().catch();
```

## 副作用与内存管理

- 优先通过 React 管理状态和副作用
- **`useEffect` 中注册的事件监听、定时器、订阅，必须在返回的清理函数中取消**

```typescript
// ✅ 正确
useEffect(() => {
    const timer = setInterval(tick, 1000);
    return () => clearInterval(timer);
}, []);
```

## 性能与资源管理

- **高频事件回调必须使用节流或防抖**，按需引入 `lodash/throttle` 或 `lodash/debounce`
- **禁止 `import { throttle } from 'lodash'` 整体引入**

```typescript
// ✅ 正确
import throttle from 'lodash/throttle';

// ❌ 错误
import { throttle } from 'lodash';
```

- 避免在渲染关键路径执行长时间同步计算
- 控制分包体积，禁止在主包中引入仅分包使用的大体积依赖

## 安全规范

| 场景 | 要求 |
|------|------|
| XSS 防护 | 禁止将不可信数据通过 `innerHTML` 注入；`rich-text` 须转义 |
| 敏感信息 | 不在代码中硬编码 Token、私钥 |
| 持久存储 | 避免保存明文敏感数据（手机号、身份证） |
| Storage Key | 通过 `cStorageKeys` 常量引用 |

---

## TypeScript 使用规范

| 规则 | 要求 |
|------|------|
| 禁止 `any` | 使用 `unknown` 配合类型守卫 |
| 禁止 `@ts-ignore` / `@ts-nocheck` | 严格类型检查 |
| 组件 props | 必须用 `interface` 或 `type` 显式声明 |
| 包装类型 | 禁止 `String`、`Number`、`Boolean`、`Object`、`Function`，使用原始类型 |

业务状态、枚举值集中定义在 `src/consts/`。

---

## 日志规范

**禁止裸露的 `console.log` / `console.warn` / `console.error`**，统一使用 `hLogger`。

```typescript
// ✅ 正确
hLogger.error({ desc: '商品详情加载失败', error, goodsId });
hLogger.info({ desc: '用户进入页面', page: 'GoodsDetail' });

// ❌ 错误
console.log('start refund', res);
console.error('error', e);
```

`hLogger` 支持 `debug` / `info` / `warn` / `error` / `js` / `api` 等级别。临时调试代码开发完成后必须删除。
