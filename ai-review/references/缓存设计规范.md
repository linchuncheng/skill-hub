# 缓存设计规范

指导团队合理使用缓存（Redis、Caffeine、自研组件）。

---

## 总体目标

- **性能提升**: 减轻数据库压力
- **行为可控**: 过期策略清晰
- **风险可控**: 防止穿透、击穿、雪崩
- **可维护**: 便于审查

---

## 适用范围

- Redis 分布式缓存
- Caffeine 本地缓存
- 自研组件：`MyRedis`、`@MyCacheLoad`、`@MyBatchKeyCacheLoad`

---

## Key 设计

### 命名规范

- 格式：`{业务前缀}::{业务维度}::{参数}`
- 示例：`user_profile::id::123`、`product_stock::sku::10001`

### 注解缓存约定

| 注解 | 说明 |
|-----|------|
| `@MyCacheLoad` | `name` 全局唯一，`suffix` 用 `{param}` 占位 |
| `@MyBatchKeyCacheLoad` | `name`+`param`+`suffix` 构成批量模板 |

---

## 过期时间与容量

| 规范 | 要求 |
|-----|------|
| **必须设置 TTL** | 所有写入操作显式设置过期时间 |
| **TTL ≥ 30秒** | 一般业务数据，空值缓存除外 |
| **本地缓存设容量** | Caffeine 用 `maximumSize()` 限制 |
| **过期抖动** | 热点 Key 加随机值防雪崩 |

---

## 风险防护

| 风险 | 防护措施 |
|-----|---------|
| **穿透** | 空结果写入空值缓存，TTL 30-120秒 |
| **击穿** | 热点 Key 加锁（本地/分布式），单线程回源 |
| **雪崩** | 过期时间加随机值（±10%） |

> 使用 `@MyCacheLoad`/`@MyBatchKeyCacheLoad` 时，组件已封装防护逻辑

---

## 批量操作规范

| 场景 | 规范 |
|-----|------|
| **禁止循环读** | ❌ `for` 内调用 `redis.get()`，改用 `mget` |
| **推荐批量读** | ✅ 先 `mget` 缓存，再批量查库回填 |
| **写操作** | 允许循环，但评估压力，可用 pipeline 优化 |
| **批量删除** | Key > 100 时分片（每 50 个一组） |

---

## 自研组件规范

### `MyRedis`

- 写入必须传 `millis` 过期时间
- 优先用批量方法（`mGet`、`delete(List)`）

### `@MyCacheLoad`

| 属性 | 要求 |
|-----|------|
| `name` | 唯一且有业务含义 |
| `expire`+`timeUnit` | 必须显式设置 |
| `cacheType` | 指定 REMOTE/LOCAL/BOTH |
| `cacheNullValue` | 建议开启防穿透 |
| `invokeLockPolicy` | 热点 Key 用锁策略 |

### `@MyBatchKeyCacheLoad`

- `param` 指定 List 类型核心参数
- 与 `@MyBatchKeyCacheEvict` 保持 `cacheType` 一致
- **内部必须用 IN 查询，禁止循环查库**

### 失效注解

| 注解 | 时机 | 场景 |
|-----|------|------|
| `@MyCacheEvict` | - | 单键失效 |
| `@MyBatchKeyCacheEvict` | 方法**前** | 极高一致性要求 |
| `@MyBatchKeyCacheEvictAfter` | 方法**后** | ✅ **推荐**，配合事务提交 |

---

## 审查关注点

- [ ] 禁止循环单 Key 读
- [ ] 统一设置过期时间
- [ ] 空值缓存防穿透
- [ ] 热点 Key 加锁防击穿
- [ ] 本地缓存设容量上限
- [ ] 自研组件参数合规

## 附录：示例

### 1. 循环读 → 批量读

```java
// ❌ 反例：循环单条读
for (String id : ids) {
    redisTemplate.opsForValue().get("product::" + id);
}

// ✅ 正例：批量读
List<String> keys = ids.stream().map(id -> "product::" + id).collect(Collectors.toList());
List<Object> values = redisTemplate.opsForValue().multiGet(keys);
```

### 2. 必须设置 TTL

```java
// ❌ 反例：无过期时间
redisTemplate.opsForValue().set(key, product);

// ✅ 正例：显式设置 TTL
redisTemplate.opsForValue().set(key, product, 60, TimeUnit.MINUTES);
```

### 3. 穿透与击穿防护

```java
public Product getById(String id) {
    String key = "product::" + id;
    Product cached = (Product) redisTemplate.opsForValue().get(key);
    if (cached != null) return cached.equals(EMPTY) ? null : cached;
    
    // 加锁防击穿
    Lock lock = myRedis.getLock("lock:" + key);
    if (!lock.tryLock(200, 5000)) return null;
    
    try {
        // 双检
        cached = (Product) redisTemplate.opsForValue().get(key);
        if (cached != null) return cached.equals(EMPTY) ? null : cached;
        
        Product db = productRepository.findById(id);
        if (db == null) {
            redisTemplate.opsForValue().set(key, EMPTY, 60, TimeUnit.SECONDS); // 空值防穿透
            return null;
        }
        redisTemplate.opsForValue().set(key, db, 60, TimeUnit.MINUTES);
        return db;
    } finally {
        lock.unLock();
    }
}
```
