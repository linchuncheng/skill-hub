# Diagram - 绘制技术图

## 核心能力

| 能力 | 说明 | 生成方式 |
|------|------|----------|
| 系统架构图 | 展示系统分层、组件、依赖关系 | LLM生成 |
| 调用链路图 | 展示服务间调用关系、通信协议 | LLM生成 |
| 部署架构图 | 展示物理/云部署拓扑 | LLM生成 |
| 数据流图 | 展示数据流转、处理过程 | LLM生成 |
| 设计原理图 | 展示技术组件内部机制、执行流程、状态变化 | LLM生成 |
| 时序图 | 展示对象间交互的时间顺序 | LLM生成 |
| 状态图 | 展示状态流转关系 | LLM生成 |
| 流程图 | 展示业务流程或算法流程 | LLM生成 |
| 泳道图 | 按角色/系统划分流程步骤 | LLM生成 |
| 领域模型图 | 展示领域模型及关联关系 | 脚本生成 |

## 生成方式

### 模式一：LLM生成

基于 `LLM` 自动生成，适用于系统架构图、调用链路图等，需要灵活布局和自定义样式的场景。

详细规范见 [references/llm-svg.md](references/llm-svg.md)

#### 设计原理图规范

设计原理图用于展示**单个技术组件的内部机制**，包括执行流程、状态变化、数据流转、决策分支等。

**适用场景：**

**1. 现有技术机制解析**
- MQ 消息队列的执行机制（生产者→Exchange→Queue→消费者）
- 网关路由策略（鉴权→限流→路由→转发）
- 缓存一致性协议（读取→命中判断→回源→回填）
- 分布式事务（Try→Confirm/Cancel）
- 状态机（订单状态流转）
- 数据库索引工作原理（B+树、查询优化器）
- JVM 垃圾回收机制（G1/ZGC 的标记-复制-清理流程）

**2. 设计方案核心亮点表达**
- 创新架构的核心机制（如：Serverless 冷启动优化流程）
- 性能优化方案（如：多级缓存穿透保护策略）
- 高可用设计（如：故障自动发现→隔离→恢复流程）
- 安全机制（如：零信任架构的认证鉴权流程）
- 数据一致性方案（如：最终一致性的补偿机制）
- 成本控制策略（如：弹性伸缩的决策算法）

**3. 算法/策略可视化**
- 负载均衡算法（加权轮询、一致性哈希的节点选择过程）
- 限流熔断策略（令牌桶、滑动窗口的计数逻辑）
- 数据分片策略（Range/Hash/一致性哈希的映射规则）
- 调度算法（任务优先级、资源分配的决策流程）

**视觉元素：**
| 元素 | 形状 | 用途 |
|------|------|------|
| 组件/模块 | 圆角矩形 | 功能模块、服务组件 |
| 决策点 | 菱形 | 条件判断、分支选择 |
| 状态 | 圆形/椭圆 | 状态机中的状态 |
| 数据/消息 | 平行四边形 | 数据包、消息体 |
| 箭头 | 带箭头线条 | 流向、顺序、依赖 |
| 分层背景 | 浅色矩形框 | 逻辑分层（接入层/逻辑层/存储层）|

**颜色规范：**
- 主流程：蓝色 (#3b82f6)
- 成功路径：绿色 (#10b981)
- 失败/异常：红色 (#ef4444)
- 可选/分支：灰色 (#9ca3af)
- 背景分层：浅灰 (#f3f4f6)

**直接描述示例：**
```
/diagram 绘制设计原理图 "RocketMQ 消息发送机制，包括生产者、NameServer、Broker、
消息刷盘策略（同步/异步）、消费者拉取消息的过程"
```

**基于文档生成示例：**

# 基于飞书文档

## 使用方法

### 命令格式

**方式一：直接描述**
```
/diagram 绘制[类型]图 [描述]
```

**方式二：基于文档生成（支持设计原理图）**
```
/diagram 绘制设计原理图 基于文档 [文档路径] 场景 [场景名称/编号]
/diagram 绘制设计原理图 基于文档 [文档路径] 场景 "场景1,场景3,场景5"
```

### 参数说明

| 参数 | 可选值 | 说明 |
|------|--------|------|
| 类型 | 系统/调用链路/部署/数据流/设计原理/时序/状态/流程/泳道/领域模型 | 绘图类型 |
| 描述 | 任意文本 | 主题描述（方式一）|
| 文档路径 | 文件路径或URL | 设计文档/需求文档路径（方式二）|
| 场景 | 场景名称或编号 | 指定文档中的具体场景，多个用逗号分隔 |

### 基于文档生成的工作流程

1. **读取文档**：解析用户提供的设计文档或需求文档（支持 Markdown、Word、PDF、飞书文档链接等）
2. **场景识别**：自动识别文档中需要可视化的技术场景（如：核心流程、异常处理、状态流转等）
3. **场景提取**：根据用户指定的场景名称/编号，提取对应的技术细节
4. **生成图表**：基于提取的内容生成设计原理图

### 基于文档生成的示例

```
# 基于本地文档生成
/diagram 绘制设计原理图 基于文档 /Users/jensen/Documents/订单系统设计.md 场景 "下单流程"

# 基于飞书文档生成
/diagram 绘制设计原理图 基于文档 https://feishu.cn/docx/xxx 场景 "支付回调处理,库存扣减机制"

# 基于需求文档生成多个场景
/diagram 绘制设计原理图 基于文档 /Users/jensen/Downloads/PRD.md 场景 "场景1,场景2,场景3"
```

## 输出规范

### 文件输出

| 项目 | 规范 |
|------|------|
| 格式 | SVG |
| 路径 | `/{图名称}.svg` |
| 编码 | UTF-8 |

### 文本语言策略

- **默认语言**：描述性信息优先使用中文
- **技术标识**：类名、方法名、表名等保持英文/代码风格
- **落盘兼容**：若存在编码风险，同时输出中文版和英文版
