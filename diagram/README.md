# Diagram - 技术图表绘制

一句话自然语言生成架构图、流程图、时序图、领域模型图等动态 SVG 技术图表。

## 使用方法

```
/diagram 绘制[类型]图 [描述]
```

**示例**：
- /diagram 绘制电商架构图
- /diagram 根据xxx.sql生成电商领域模型图
- /diagram 结合@README.md生成订单售后时序图
- /diagram 绘制Redis缓存核心原理图

## 支持的图表类型（10种）

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| 系统架构图 | 系统分层、组件、依赖关系 | 分层架构、组件依赖 |
| 调用链路图 | 服务间调用关系、通信协议 | 服务调用、微服务拓扑 |
| 部署架构图 | 物理/云部署拓扑 | 服务器、容器、云资源 |
| 数据流图 | 数据流转、处理过程 | ETL、消息流转 |
| 时序图 | 对象间交互的时间顺序 | API 调用流程、消息序列 |
| 状态图 | 状态流转关系 | 订单状态、审批流程 |
| 流程图 | 业务流程或算法流程 | 业务逻辑、算法步骤 |
| 泳道图 | 按角色/系统划分流程步骤 | 跨部门协作流程 |
| 核心原理图 | 技术组件内部机制、执行流程 | Redis 缓存机制、消息队列原理 |
| 领域模型图 | 领域模型及关联关系 | DDD 领域模型、数据库 ER 图 |

## 三种工作流

| 工作流 | 适用图类型 | 核心步骤 |
|--------|-----------|----------|
| **标准工作流** | 系统架构图、调用链路图、部署架构图、数据流图、时序图、状态图、流程图、泳道图 | 需求分析 → ASCII布局 → SVG布局 → 绘制 → 连线 → 动画 → 验证 → 修复 → 输出 |
| **核心原理图专属** | 核心原理图 | 核心思考框架(4问) → 布局意图 → SVG布局 → 有机绘制 → 验证 → 输出 |
| **领域模型图专属** | 领域模型图 | 分析SQL → 领域划分 → 四色类型 → ASCII布局 → 生成JSON → 脚本生成SVG → 预览 |

**关键约束**：❌ 禁止跳过布局规划  ❌ 禁止跳过验证(`llm_svg_validator.py`)  ❌ 验证失败必须修复(最多3次)

## 目录结构

```
diagram/
├── SKILL.md                          # 核心工作流定义
├── manifest.json                     # 技能元数据
├── assets/                           # 10种SVG示例图 + 领域模型JSON/SQL
├── scripts/                          # 验证器、生成器
└── references/
    ├── svg-common.md                 # 公共规范（配色、动画、布局、约束）
    ├── line_standard.md              # 连线规范权威来源（四边中点系统、箭头）
    ├── {图类型}-svg.md               # 各图类型专属规范
    ├── principle-svg.md              # 核心原理图专属规范
    └── model-svg.md                  # 领域模型图专属规范
```

## 输出规范

| 项目 | 规范 |
|------|------|
| 格式 | SVG（动态，含动画） |
| 路径 | `/{图名称}.svg` |
| 编码 | UTF-8 |
| 文本语言 | 描述性信息用中文，技术标识用英文 |
