# 调用链路图规范

## 结构

```
发起方 → 中间层 → 处理方 → 下游依赖
```

## 节点类型

| 类型 | 说明 | 图形 |
|------|------|------|
| 发起方 | 请求的起始点 | 圆角矩形 |
| 中间层 | 转发/处理节点 | 矩形 |
| 处理方 | 核心业务处理 | 矩形 |
| 下游依赖 | 数据库/外部服务 | 圆柱/云形 |

## 绘制要点

### 布局原则
- 采用**从左到右**的水平布局
- 同一调用层级的节点垂直对齐
- 并行调用使用分支结构

### 连线规则
- 同步调用：实线箭头
- 异步调用：虚线箭头
- 回调：带标签的返回线
- 超时/重试：虚线 + 标注

### 标注规范
- 连线上标注接口名称
- 关键参数可在连线上方标注
- 耗时信息可在连线下方标注

## 示例场景

- 下单流程链路
- 支付回调链路
- 数据同步链路
- 消息推送链路

## SVG 模板

```xml
<!-- 调用链路图基础结构 -->
<g>
  <!-- 发起方 -->
  <rect x="x" y="y" width="W" height="H" rx="8" fill="#fff9c4" stroke="#f9a825"/>
  
  <!-- 中间层 -->
  <rect ... fill="#e3f2fd" stroke="#1976d2"/>
  
  <!-- 处理方 -->
  <rect ... fill="#f1f8e9" stroke="#689f38"/>
  
  <!-- 下游依赖 -->
  <ellipse ... fill="#fce4ec" stroke="#e91e63"/>
  
  <!-- 连线 -->
  <line x1="x1" y1="y1" x2="x2" y2="y2" stroke="#666" marker-end="url(#arrow)"/>
  <text>接口名</text>
</g>
```
