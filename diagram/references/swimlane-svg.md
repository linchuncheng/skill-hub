# 泳道图规范

> 📋 **工作流规范**：泳道图的专属工作流定义在 [SKILL.md](../SKILL.md) 中，执行前请先阅读。
>
> **专属工作流**：需求分析 → 生成JSON配置 → 执行脚本生成SVG → 预览调整
>
> **核心原则**：LLM 只需生成准确的 JSON 配置，连线、布局、配色等视觉细节由脚本自动处理

## JSON配置

### JSON 格式

```json
{
  "title": "图表标题",
  "columns": ["用户", "商城系统", "支付系统"],
  "rows": ["下单阶段", "支付阶段"],
  "steps": [
    {"label": "提交订单:选择商品", "col": 1, "row": 1},
    {"label": "创建订单", "col": 2, "row": 1}
  ],
  "connections": [
    {"from": "提交订单", "to": "创建订单"},
    {"from": "创建订单", "to": "发起支付", "type": "async", "label": "待支付"}
  ]
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `title` | string | 是 | 图表标题，格式为「XXX流程」而非「XXX泳道图」 |
| `columns` | string[] | 是 | 纵向泳道名称（角色/系统），从左到右排列 |
| `rows` | string[] | 否 | 横向泳道名称（阶段），从上到下排列；简单流程可不设 |
| `steps[].label` | string | 是 | `主标题` 或 `主标题:副标题`，用冒号分隔；**主标题建议≤13字，副标题建议≤15字，避免超出组件** |
| `steps[].col` | int | 是 | 所在列序号（从1开始），对应 columns |
| `steps[].row` | int | 是 | 所在行序号（从1开始），对应 rows |
| `connections[].from` | string | 是 | 起点步骤的主标题 |
| `connections[].to` | string | 是 | 终点步骤的主标题 |
| `connections[].type` | string | 否 | `sync`（默认，同步调用）或 `async`（异步消息） |
| `connections[].label` | string | 否 | 连线标签文字 |

### 编排规则

| 规则 | 说明 |
|------|------|
| 主标题唯一 | 每个 step 的主标题必须唯一，connections 通过主标题引用 |
| from/to 精确匹配 | connections 中的 from/to 必须与某个 step 的主标题完全一致，否则连线不显示 |
| 流程方向 | 连线尽量从上到下、从左到右，脚本会自动处理反向和跨行连线 |

### 编排示例

以电商下单流程为例，先确定列（角色）和行（阶段），再填入步骤：

```
columns: ["用户", "商城系统", "支付系统", "履约系统", "结算系统"]
rows: ["下单阶段", "支付阶段", "履约阶段", "结算阶段"]

  用户       商城系统         支付系统       履约系统       结算系统
  ┌─────────┬───────────────┬─────────────┬─────────────┬──────────┐
  │提交订单  │创建订单        │             │             │          │ 下单阶段
  │         │锁定库存        │             │             │          │
  ├─────────┼───────────────┼─────────────┼─────────────┼──────────┤
  │发起支付  │支付回调        │扣款处理      │             │          │ 支付阶段
  ├─────────┼───────────────┼─────────────┼─────────────┼──────────┤
  │确认收货  │发货通知        │             │拣货打包      │          │ 履约阶段
  ├─────────┼───────────────┼─────────────┼─────────────┼──────────┤
  │         │               │             │发起结算      │分账计算   │ 结算阶段
  │         │               │             │             │资金划拨   │
  └─────────┴───────────────┴─────────────┴─────────────┴──────────┘
```

## JSON配置示例

```json
{
  "title": "电商下单履约结算流程",
  "columns": ["用户", "商城系统", "支付系统", "履约系统", "结算系统"],
  "rows": ["下单阶段", "支付阶段", "履约阶段", "结算阶段"],
  "steps": [
    {"label": "提交订单:", "col": 1, "row": 1},
    {"label": "创建订单:生成订单号/计算金额", "col": 2, "row": 1},
    {"label": "锁定库存:预占可用库存", "col": 2, "row": 1},
    {"label": "发起支付:", "col": 1, "row": 2},
    {"label": "支付回调:验证签名/更新状态", "col": 2, "row": 2},
    {"label": "扣款处理:冻结资金/扣减余额", "col": 3, "row": 2},
    {"label": "发货通知:", "col": 2, "row": 3},
    {"label": "拣货打包:打印面单/分配快递", "col": 4, "row": 3},
    {"label": "确认收货:", "col": 1, "row": 3},
    {"label": "发起结算:触发分账流程", "col": 4, "row": 4},
    {"label": "分账计算:佣金/运费/分润", "col": 5, "row": 4},
    {"label": "资金划拨:打款到商家账户", "col": 5, "row": 4}
  ],
  "connections": [
    {"from": "提交订单", "to": "创建订单"},
    {"from": "创建订单", "to": "锁定库存"},
    {"from": "创建订单", "to": "发起支付", "type": "async", "label": "待支付"},
    {"from": "发起支付", "to": "支付回调"},
    {"from": "支付回调", "to": "扣款处理"},
    {"from": "扣款处理", "to": "发货通知", "type": "async", "label": "支付成功"},
    {"from": "发货通知", "to": "拣货打包"},
    {"from": "拣货打包", "to": "确认收货", "type": "async", "label": "物流签收"},
    {"from": "确认收货", "to": "发起结算", "type": "async", "label": "待结算"},
    {"from": "发起结算", "to": "分账计算"},
    {"from": "分账计算", "to": "资金划拨"}
  ]
}
```
