# 组件连线规范 - Line Standard

> **适用范围**：所有类型的 SVG 图（架构图、原理图、流程图、时序图等）的组件连线
> 
> **优先级**：本规范为连线规范的唯一权威来源，其他文档中的连线规范如与本规范冲突，以本规范为准。

## 核心原则

组件间连线必须严格遵循以下规范，确保连线**清晰、规范、美观**。

### ⚠️ 关键经验教训（必读！）

**1. 箭头尺寸问题**
- ❌ 错误：使用 `<polygon>` 绘制箭头，尺寸过大（10x7），颜色固定
- ✅ 正确：使用 `<path>` 绘制实心彩色凹口箭头，viewBox 8x8，显示 5x5
- 📍 标准定义见第 12.1 节

**2. 连线计算问题**
- ❌ 错误：LLM 手动估算连接点坐标和路径
- ✅ 正确：使用 `model_line_generator.py` 的 `get_best_path()` 自动计算
- 📍 工具使用见第 9 节

**3. 圆角风格问题**
- ❌ 错误：部分连线用 `L` 命令（直角），部分用 `Q` 命令（圆角）
- ✅ 正确：**原理图/概念图必须全图使用圆角风格（Q 命令）**
- 📍 风格选择见第 3.1 节

**4. 四边中点问题**
- ❌ 错误：手动估算连接点，导致连线不从中心出发
- ✅ 正确：严格使用公式计算：`L(x1, (y1+y2)/2)`, `R(x2, (y1+y2)/2)`, `T((x1+x2)/2, y1)`, `B((x1+x2)/2, y2)`
- 📍 计算公式见第 1.1 节

## 1. 连接点：四边中点系统

### 1.1 四边中点定义

每个矩形组件有 **4 个合法连接点**，均为边的中点：

```
                T (top-middle)
                (center_x, y1)
              ┌─────────────────┐
              │                 │
  L (left-mid)│                 │R (right-mid)
(x1, center_y)│                 │(x2, center_y)
              │                 │
              └─────────────────┘
               B (bottom-middle)
                (center_x, y2)

计算公式：
  L = (x1, (y1+y2)/2)     # 左中点
  R = (x2, (y1+y2)/2)     # 右中点
  T = ((x1+x2)/2, y1)     # 上中点
  B = ((x1+x2)/2, y2)     # 下中点
```

### 1.2 连接点选择策略

**基本原则：选择距离最近的两个中点连接**

1. 计算起点组件 4 个中点 到 终点组件 4 个中点的欧几里得距离（共 16 种组合）
2. 按距离从小到大排序
3. 从最近的组合开始尝试，找到第一条可行路径即停止

**示例**：
```
组件 A: x=100-200, y=50-100
  L(100, 75), R(200, 75), T(150, 50), B(150, 100)

组件 B: x=300-400, y=150-200
  L(300, 175), R(400, 175), T(350, 150), B(350, 200)

距离计算（部分）：
  A.R(200,75) → B.L(300,175): √((300-200)² + (175-75)²) = 141.4  ← 最近
  A.B(150,100) → B.T(350,150): √((350-150)² + (150-100)²) = 206.2
  ...

优先尝试：A.R → B.L
```

## 2. 方向约束：垂直于边框

### 2.1 出发方向约束

从组件某边出发时，**第一段线段必须垂直于该边**：

| 出发边 | 线段方向 | 坐标变化 | 示例 |
|--------|----------|----------|------|
| **L 边** | 水平向左 | x 减小，y 不变 | `(100, 75) → (80, 75)` ✓ |
| **R 边** | 水平向右 | x 增大，y 不变 | `(200, 75) → (220, 75)` ✓ |
| **T 边** | 垂直向上 | y 减小，x 不变 | `(150, 50) → (150, 30)` ✓ |
| **B 边** | 垂直向下 | y 增大，x 不变 | `(150, 100) → (150, 120)` ✓ |

**错误示例**：
```
❌ L 边出发却向右：(100, 75) → (120, 75)  # 进入组件内部了
❌ T 边出发却向下：(150, 50) → (150, 70)  # 进入组件内部了
❌ L 边出发却垂直：(100, 75) → (100, 95)  # 不垂直于 L 边
```

### 2.2 进入方向约束

进入组件某边时，**最后一段线段必须垂直于该边**：

| 进入边 | 线段方向 | 坐标变化 | 示例 |
|--------|----------|----------|------|
| **L 边** | 从左向右进入 | x 增大，y 不变 | `(80, 75) → (100, 75)` ✓ |
| **R 边** | 从右向左进入 | x 减小，y 不变 | `(220, 75) → (200, 75)` ✓ |
| **T 边** | 从上向下进入 | y 增大，x 不变 | `(150, 30) → (150, 50)` ✓ |
| **B 边** | 从下向上进入 | y 减小，x 不变 | `(150, 120) → (150, 100)` ✓ |

**记忆口诀**：
- L 边：出发往左（←），进入从右来（→）
- R 边：出发往右（→），进入从左来（←）
- T 边：出发往上（↑），进入从下来（↓）
- B 边：出发往下（↓），进入从上来（↑）

## 3. 线段约束：90° 折线

### 3.1 连线风格选择（重要！）

**核心原则：同一张图中，所有连线必须统一使用一种风格**

| 风格 | 命令 | 特点 | 适用场景 |
|------|------|------|----------|
| **直角风格** | 仅使用 `M` + `L` 命令 | 折线清晰，技术感强 | 架构图、流程图、数据流图 |
| **圆角风格** | 使用 `M` + `L` + `Q` 命令 | 流畅柔和，视觉友好 | **原理图、领域模型图、概念图** |

**⚠️ 重要说明**：
1. **原理图/概念图/领域模型图** → **必须使用圆角风格**（推荐）
2. **架构图/流程图/数据流图** → 推荐使用直角风格
3. **一旦选定，全图统一**，禁止混用

**圆角风格实现方式**：
```python
# 使用 model_line_generator.py 自动生成
from model_line_generator import ComponentLineGenerator

gen = ComponentLineGenerator()
path = gen.get_best_path(start_comp, end_comp, all_components)
d = gen.get_rounded_svg_path(path)  # 自动生成带圆角的 SVG path
```

**严格禁止**：
- ❌ 同一张图中混合使用直角和圆角
- ❌ 同一条连线中 `L` 和 `Q` 命令混用（圆角风格允许在直线段使用 L，拐点使用 Q）
- ❌ 部分连线用直角，部分连线用圆角

**选择指南**：
1. **架构图/流程图** → 推荐直角风格（L 命令）
2. **原理图/概念图** → 推荐圆角风格（Q 命令）
3. **一旦选定，全图统一**

### 3.2 直角风格规范（使用 L 命令）

- ✅ 所有线段必须是**水平或垂直**的
- ✅ 相邻线段必须**交替**（水平→垂直→水平...）
- ✅ 转角必须是 **90°**（不允许斜线或任意角度）

### 3.3 圆角风格规范（使用 model_line_generator.py）

**⚠️ 核心要求：圆角连线必须使用 `model_line_generator.py` 的 `get_rounded_svg_path()` 方法生成**

**为什么不能手动绘制圆角？**
1. 圆角计算复杂：需要计算单位向量、实际圆角半径、拐点前后位置
2. 容易出错：手动计算容易导致弧度过大/过小、方向错误
3. 难以维护：调整组件位置后需要重新计算所有圆角

**正确工作流程**：
```python
# 1. 定义组件坐标
components = [
    {'model_id': 'A', 'x1': 100, 'y1': 50, 'x2': 200, 'y2': 100},
    {'model_id': 'B', 'x1': 300, 'y1': 150, 'x2': 400, 'y2': 200},
]

# 2. 计算最优路径（自动避障）
gen = ComponentLineGenerator()
path = gen.get_best_path(components[0], components[1], components)

# 3. 生成带圆角的 SVG path
d = gen.get_rounded_svg_path(path, radius=8.0)

# 4. 输出到 SVG
print(f'<path d="{d}" stroke="#3b82f6" stroke-width="2" fill="none" marker-end="url(#arrow)"/>')
```

**圆角半径配置**：
- 标准圆角：`radius=8.0`（`CORNER_RADIUS` 默认值）
- 紧凑布局：`radius=5.0`（组件间距 < 50px）
- 宽松布局：`radius=10.0`（组件间距 > 100px）

**圆角连线模板**（仅供参考，实际应使用脚本生成）：
```xml
<!-- 水平直线段 -->
<path d="M x1 y L x2 y" .../>

<!-- 垂直直线段 -->
<path d="M x y1 L x y2" .../>

<!-- 90° 圆角转折 -->
<path d="M x1 y1 L corner_start_x corner_start_y Q拐点x 拐点y corner_end_x corner_end_y L x2 y2" .../>
```

**关键规则**：
1. ✅ 必须使用 `model_line_generator.py` 生成路径（不要手动计算）
2. ✅ 圆角半径默认 8px（可在 `CORNER_RADIUS` 中调整）
3. ✅ 直线段使用 `L` 命令，拐点处使用 `Q` 命令
4. ✅ 圆角自动适配线段长度（不超过线段一半）

### 3.4 最小线段长度

**任何线段在转折前，长度必须 ≥ 20px**

```
正确示例：
  (100, 75) → (80, 75) → (80, 150)
  第一段长度 = |100-80| = 20px  ✓ 刚好满足

错误示例：
  (100, 75) → (90, 75) → (90, 150)
  第一段长度 = |100-90| = 10px  ✗ 太短就转折
```

**计算方法**：
```python
# 水平线段长度
length = abs(x2 - x1)

# 垂直线段长度
length = abs(y2 - y1)

# 验证
if length < 20:
    # 需要延长线段或选择其他路径
```

## 4. 路径生成：按折线数递增

### 4.1 折线数 = 0（直连）

**适用条件**：两个组件**同行或同列**，且中间无阻挡

```
同行（y 中心相近）：
  A.R → B.L  水平直线
  A.L → B.R  水平直线

同列（x 中心相近）：
  A.B → B.T  垂直直线
  A.T → B.B  垂直直线

示例：
  A(100-200, 50-100) → B(250-350, 50-100)
  路径：M 200 75 L 250 75  （1 段线段，0 折线）
```

### 4.2 折线数 = 1（L 形，1 个拐点）

**适用条件**：不同类型的边连接（水平边 ↔ 垂直边）

| 出发边 | 进入边 | 路径形状 | 拐点计算 |
|--------|--------|----------|----------|
| L/R | T | └ 形 | `(ex, sy)` |
| L/R | B | ┌ 形 | `(ex, sy)` |
| T | L/R | ┐ 形 | `(sx, ey)` |
| B | L/R | ┘ 形 | `(sx, ey)` |

**示例 1**：A.T → B.L
```
A: (100-200, 50-100), T 中点 (150, 50)
B: (300-400, 150-200), L 中点 (300, 175)

路径：先垂直向下，再水平向右
拐点：(sx, ey) = (150, 175)

SVG: M 150 50 L 150 175 L 300 175
验证：
  第 1 段：|175-50| = 125px ≥ 20 ✓
  第 2 段：|300-150| = 150px ≥ 20 ✓
```

**示例 2**：A.R → B.T
```
A: (100-200, 50-100), R 中点 (200, 75)
B: (300-400, 150-200), T 中点 (350, 150)

路径：先水平向右，再垂直向下
拐点：(ex, sy) = (350, 75)

SVG: M 200 75 L 350 75 L 350 150
验证：
  第 1 段：|350-200| = 150px ≥ 20 ✓
  第 2 段：|150-75| = 75px ≥ 20 ✓
```

### 4.3 折线数 = 2（Z/U 形，2 个拐点）

**适用条件**：相同类型的边连接（水平边 ↔ 水平边，或垂直边 ↔ 垂直边）

#### 情况 A：L/R → L/R（水平→垂直→水平）

```
出发边方向：
  L 边：往左（x 减小）
  R 边：往右（x 增大）

进入边方向：
  L 边：从右来（x 增大）
  R 边：从左来（x 减小）

路径模板：
  拐点 1: (mid_x, sy)  # 先水平移动
  拐点 2: (mid_x, ey)  # 再垂直移动

mid_x 选择策略：
  1. 计算 sx 和 ex 的中点：(sx + ex) / 2
  2. 确保 |mid_x - sx| ≥ 20（第 1 段长度）
  3. 确保 |mid_x - ex| ≥ 20（第 3 段长度）
  4. 如果中点不满足，尝试偏移 ±50、±100、±150

示例：A.L(100, 75) → B.L(300, 175)
  sx=100, sy=75, ex=300, ey=175
  mid_x 候选：
    中点：(100+300)/2 = 200
    验证：|200-100|=100 ≥ 20 ✓, |200-300|=100 ≥ 20 ✓
  
  路径：M 100 75 L 200 75 L 200 175 L 300 175
  线段：
    1. (100,75)→(200,75): 100px ✓
    2. (200,75)→(200,175): 100px ✓
    3. (200,175)→(300,175): 100px ✓
```

#### 情况 B：T/B → T/B（垂直→水平→垂直）

```
路径模板：
  拐点 1: (sx, mid_y)  # 先垂直移动
  拐点 2: (mid_x, mid_y)  # 再水平移动
  拐点 3: (mid_x, ey)  # 最后垂直移动

注意：T/B → T/B 实际需要 3 段垂直+水平交替，
所以是 2 折线（3 段线段）

mid_y 选择策略：
  1. 计算 sy 和 ey 的中点
  2. 确保 |mid_y - sy| ≥ 20
  3. 尝试偏移 ±50、±100 找到不被阻挡的位置

mid_x 选择策略：
  1. 计算 sx 和 ex 的中点
  2. 尝试偏移 ±50、±100 避开阻挡组件

示例：A.B(150, 100) → B.T(350, 50)
  mid_y = (100+50)/2 = 75
  mid_x = (150+350)/2 = 250
  
  路径：M 150 100 L 150 75 L 250 75 L 250 50 L 350 50
  （需要验证是否被阻挡，如被阻挡则调整 mid_x/mid_y）
```

### 4.4 折线数 ≥ 3（复杂绕行）

**适用条件**：组件密集、多次阻挡需要大幅绕行

**策略**：
1. 选择绕行方向（上/下/左/右）
2. 计算绕行路径的关键点
3. 验证所有线段 ≥ 20px
4. 验证不穿过任何组件

## 5. 避让规则：不穿过组件

### 5.1 阻挡检测

线段的任何部分（包括端点）**不得进入其他组件的矩形区域**。

**检测方法**：
```python
def is_blocked(segment, component):
    (x1, y1), (x2, y2) = segment
    comp_x1, comp_y1, comp_x2, comp_y2 = component 边界
    
    # 水平线段
    if y1 == y2:
        if comp_y1 <= y1 <= comp_y2:  # y 在组件范围内
            if not (x2 < comp_x1 or x1 > comp_x2):  # x 有交集
                return True  # 被阻挡
    
    # 垂直线段
    if x1 == x2:
        if comp_x1 <= x1 <= comp_x2:  # x 在组件范围内
            if not (y2 < comp_y1 or y1 > comp_y2):  # y 有交集
                return True  # 被阻挡
    
    return False
```

### 5.2 绕行策略

当直连或简单折线被阻挡时：

1. **小幅绕行**：偏移 50px
2. **中幅绕行**：偏移 100px
3. **大幅绕行**：偏移 150-200px
4. **换边尝试**：改用其他边连接点

**示例**：
```
A.R(200, 75) → B.L(300, 75) 被 C 阻挡

尝试 1（上方绕行）：
  拐点：(250, 25), (250, 75)
  路径：M 200 75 L 250 75 L 250 25 L 300 25 L 300 75

尝试 2（下方绕行）：
  拐点：(250, 125), (250, 75)
  路径：M 200 75 L 250 75 L 250 125 L 300 125 L 300 75

选择：验证哪条不被阻挡，选折线数最少的
```

## 6. 最优路径选择

当存在多条可行路径时，按以下优先级选择：

### 6.1 优先级排序

```
1. 折线数最少（拐点少，路径简洁）
2. 曼哈顿距离最短（路径总长度短）
3. 拐点坐标更"规整"（优先选择中点位置）
```

**曼哈顿距离计算**：
```python
total_length = sum(|x2-x1| + |y2-y1| for 每段线段)
```

### 6.2 选择流程

```
for 折线数 in [0, 1, 2, 3, ...]:
    for 边组合 in 按距离排序的 16 种组合:
        生成该折线数下的候选路径
        for 每条路径:
            if 满足所有约束:
                加入可行路径列表
    
    if 可行路径列表不为空:
        选择曼哈顿距离最短的路径
        break  # 找到最优解，停止
```

## 7. LLM 计算流程

### 7.1 标准计算步骤

```
Step 1: 提取组件坐标
  - 列出所有组件的 x1, y1, x2, y2
  - 计算每个组件的 4 个中点

Step 2: 确定连接的边
  - 计算 16 种边组合的欧几里得距离
  - 从最近组合开始尝试

Step 3: 判断折线数
  - 同行/同列且无阻挡 → 0 折线
  - 不同类型边 → 尝试 1 折线
  - 同类型边 → 尝试 2 折线
  - 被阻挡 → 增加折线数或换边

Step 4: 计算拐点
  - 根据折线数和边类型，套用对应公式
  - 确保每段 ≥ 20px

Step 5: 验证路径
  ✓ 方向约束（出发/进入方向垂直于边）
  ✓ 线段约束（≥ 20px 才能转折）
  ✓ 避让约束（不穿过任何组件）
  ✓ 90° 折线（仅水平/垂直线段）

Step 6: 生成 SVG
  M sx sy L x1 y1 L x2 y2 ... L ex ey
```

### 7.2 快速检查清单

在生成连线后，逐项验证：

```
□ 连接点是四边中点吗？
□ 出发方向垂直于出发边吗？
□ 进入方向垂直于进入边吗？
□ 每段线段长度 ≥ 20px 吗？
□ 所有线段都是水平或垂直的吗？
□ 路径没有穿过任何组件吗？
□ 折线数是最少的吗？
```

## 8. 常见错误与修正

### 8.1 错误 1：连接点不在中点

```
❌ 错误：A(350, 88) → B(270, 170)  # 随意选的点
✓ 正确：A.L(350, 88) → B.T(270, 170)  # 明确是中点
```

### 8.2 错误 2：出发方向错误

```
❌ L 边出发向右：M 100 75 L 120 75  # 进入组件
✓ L 边出发向左：M 100 75 L 80 75    # 远离组件
```

### 8.3 错误 3：线段太短就转折

```
❌ 10px 就转折：M 100 75 L 90 75 L 90 150
✓ 至少 20px：   M 100 75 L 80 75 L 80 150
```

### 8.4 错误 4：路径穿过组件

```
组件 C: x=200-300, y=100-200
```

## 11. 连线类型与语义

| 连线类型 | 颜色 | 样式 | 语义 |

## 12. 箭头标记（Marker）规范

### 12.1 标准箭头定义

```xml
<defs>
  <marker id="arrow" viewBox="0 0 8 8" refX="7.5" refY="4"
          markerWidth="5" markerHeight="5" orient="auto">
    <path d="M 0 0 L 8 4 L 0 8 L 1 4 Z" fill="连线颜色"/>
  </marker>
</defs>
```

**参数说明**：
- `viewBox="0 0 8 8"`：箭头坐标系
- `refX="7.5" refY="4"`：箭头尖端参考点（距路径终点 0.5px）
- `markerWidth="5" markerHeight="5"`：箭头显示尺寸
- `orient="auto"`：自动旋转适配路径方向
- `fill="连线颜色"`：实心填充，颜色与连线一致
- `d="M 0 0 L 8 4 L 0 8 L 1 4 Z"`：带凹口的V形箭头路径

**形状示意**：
```
(0,0) ────────┐
  │           │
  │        (8,4) ← 箭头尖端
  │           │
(0,8) ────────┘
  ↑
(1,4) ← 凹口(1px)
```

### 12.2 箭头使用规则

| 规则 | 说明 | 示例 |
|------|------|------|
| **单向箭头** | 只在终点设置箭头 | `marker-end="url(#arrow)"` |
| **禁止双向** | 不要同时使用 `marker-start` 和 `marker-end` | ❌ 两端都有箭头 |
| **转折点无箭头** | 拐点处不显示箭头 | ❌ 在拐点加 `marker-mid` |
| **颜色一致** | 箭头填充色与连线 stroke 颜色相同 | 蓝线→蓝箭头，绿线→绿箭头 |
| **连线路径** | 必须使用 `<path>` 而非 `<line>` | ❌ `<line>` 不支持 marker |
| **连线填充** | 连线本身必须 `fill="none"` | `<path fill="none" stroke="..." ...>` |

**优先策略**（按顺序执行）：

2. **微调标签位置**：偏移标签，不进框/不压箭头
3. **最后才改走线**：改为绕行路径

**禁止事项**：
- ❌ 标签压在组件框内
- ❌ 标签压在箭头头部
- ❌ 多个标签相互覆盖
- ❌ 标签文字过长（≤ 10 字）

## 14. 连线动画规范

### 14.1 实线光点流动动画

**适用场景**：数据流、调用链、消息传递、主流程

#### 基础实现

```xml
<svg>
  <defs>
    <!-- 光点径向渐变 -->
    <radialGradient id="glow-dot" cx="50%" cy="50%" r="50%">
      <stop offset="0%" style="stop-color:#3b82f6;stop-opacity:1" />
      <stop offset="100%" style="stop-color:#3b82f6;stop-opacity:0" />
    </radialGradient>
    
    <style>
      /* 光点动画 */
      .flow-dot {
        opacity: 0;
        animation: dotFadeIn 0.3s ease-out forwards, dotMove 2s ease-in-out infinite;
      }
      
      @keyframes dotFadeIn {
        0% { opacity: 0; }
        100% { opacity: 1; }
      }
      
      @keyframes dotMove {
        0% { offset-distance: 0%; }
        100% { offset-distance: 100%; }
      }
    </style>
  </defs>
  
  <!-- 连线（必须用 path） -->
  <path class="connector-line" 
        d="M 100 100 L 200 100 L 200 200" 
        stroke="#3b82f6" stroke-width="2" fill="none"
        marker-end="url(#arrow)"
        style="animation: fadeIn 0.4s ease-out forwards; animation-delay: 0.3s;"/>
  
  <!-- 光点 -->
  <circle class="flow-dot" r="4" fill="url(#glow-dot)"
          style="offset-path: path('M 100 100 L 200 100 L 200 200'); 
                 animation-delay: 0.6s, 0.6s;">
  </circle>
</svg>
```

#### 关键要求

| 要求 | 说明 | 示例 |
|------|------|------|
| **必须用 `<path>`** | `<line>` 不支持 `offset-path` | ❌ `<line>` → ✅ `<path>` |
| **必须 `fill="none"`** | 防止 path 填充成黑色块 | `<path fill="none" .../>` |
| **offset-path 一致** | 值必须与 path 的 `d` 属性完全相同 | `d="M..."` = `offset-path: path('M...')` |
| **延迟层级** | 光点延迟 > 连线延迟（形成先后效果） | 连线 0.3s → 光点 0.6s |

#### 不同场景配色

| 场景 | 连线颜色 | 光点颜色 | 循环周期 | 延迟 |
|------|----------|----------|----------|------|
| **主流程** | `#3b82f6` (蓝) | `#3b82f6` | 2s | 0.6s |
| **成功路径** | `#10b981` (绿) | `#10b981` | 2s | 0.8s |
| **失败/异常** | `#ef4444` (红) | `#ef4444` | 1.5s | 1.0s |
| **异步调用** | `#9ca3af` (灰) | `#9ca3af` | 3s | 1.2s |
| **反馈/回调** | `#3b82f6` (蓝) | `#3b82f6` | 2.5s | 1.4s |

#### 光点大小建议

| 连线粗细 | 光点半径 | 说明 |
|----------|----------|------|
| `stroke-width: 1.5` | `r="3"` | 细线配小光点 |
| `stroke-width: 2` | `r="4"` | 标准配置 |
| `stroke-width: 2.5` | `r="5"` | 粗线配大光点 |

### 14.2 虚线流动动画

**适用场景**：异步消息、后台任务、心跳检测、可选分支

#### 基础实现

```xml
<svg>
  <style>
    .dash-flow {
      opacity: 0;
      animation: fadeIn 0.4s ease-out forwards, dashMove 2s linear infinite;
    }
    
    @keyframes dashMove {
      to { stroke-dashoffset: -9; }
    }
  </style>
  
  <!-- 虚线流动 -->
  <path class="dash-flow" 
        d="M 100 100 L 200 100 L 200 200" 
        stroke="#9ca3af" stroke-width="1.5" fill="none"
        stroke-dasharray="6 3"
        marker-end="url(#arrow)"
        style="animation-delay: 0.5s, 0.5s;"/>
</svg>
```

#### 无缝循环关键规则

**核心公式**：`stroke-dashoffset` 必须是 `stroke-dasharray` 周期的**整数倍**

| 配置项 | 值 | 说明 |
|--------|-----|------|
| **虚线模式** | `stroke-dasharray="6 3"` | 实线 6px + 空白 3px |
| **周期计算** | `6 + 3 = 9px` | 一个完整周期 |
| **动画偏移** | `stroke-dashoffset: -9` | 1 倍周期（推荐） |
| **备选偏移** | `-18, -27, -36...` | 2/3/4 倍周期（速度更快） |

**推荐配置**（统一标准）：
```xml
<!-- 标准配置：完美无缝循环 -->
<path stroke-dasharray="6 3" 
      style="stroke-dashoffset: -9; 
             animation: dashMove 2s linear infinite;"/>
```

#### ❌ 错误示例

```xml
<!-- 错误 1：偏移不是周期整数倍 → 跳变！ -->
<path stroke-dasharray="6 3" style="stroke-dashoffset: -8"/>
<!-- 周期=9，偏移=-8，不是整数倍 → 动画结束时跳变 -->

<!-- 错误 2：分段独立的线 → 重置感！ -->
<line x1="100" y1="100" x2="150" y2="100" stroke-dasharray="6 3"/>
<line x1="150" y1="100" x2="150" y2="200" stroke-dasharray="6 3"/>
<line x1="150" y1="200" x2="200" y2="200" stroke-dasharray="6 3"/>
<!-- 3 段独立动画，每段重置 → 流动不连续 -->
```

#### ✅ 正确示例

```xml
<!-- 正确 1：整数倍偏移 → 完美无缝 -->
<path stroke-dasharray="6 3" style="stroke-dashoffset: -9"/>

<!-- 正确 2：连续路径 → 流动无断点 -->
<path d="M 100 100 L 150 100 L 150 200 L 200 200" 
      stroke-dasharray="6 3" 
      style="stroke-dashoffset: -9;"/>
```

#### 虚线周期参考表

| stroke-dasharray | 周期 | 推荐 offset | 适用场景 |
|------------------|------|-------------|----------|
| `"6 3"` | 9px | `-9` | **标准推荐**，大多数场景 |
| `"5 3"` | 8px | `-8` | 较密虚线 |
| `"4 2"` | 6px | `-6` | 密集虚线 |
| `"8 4"` | 12px | `-12` | 稀疏虚线 |

## 总结

LLM 遵循此规范可计算出 **90%+** 的正确连线。核心要点：

1. **连接点**：只用四边中点（L/R/T/B）
2. **方向**：出发/进入必须垂直于边框
3. **长度**：每段线段 ≥ 20px 才能转折
4. **角度**：只能 90° 折线（水平/垂直交替）
5. **避让**：路径不能穿过任何组件
6. **最优**：优先折线数少，其次路径短
7. **样式**：使用 `<path>` + `fill="none"` + `marker-end`
8. **标注**：标签不压组件、不压箭头

遇到复杂场景时，**运行脚本计算**比人工计算更可靠、更高效。
