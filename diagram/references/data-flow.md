# 数据流图规范

## 结构

```
数据源 → 处理节点 → 存储 → 消费端
```

## 节点类型

| 类型 | 说明 | 图形 |
|------|------|------|
| 数据源 | 数据产生方 | 平行四边形 |
| 处理节点 | 数据转换/计算 | 矩形 |
| 存储 | 数据持久化 | 圆柱形/开口矩形 |
| 消费端 | 数据使用方 | 矩形 |

## 绘制要点

### 布局原则
- 采用**从左到右**的水平布局
- 数据流向清晰可见
- 分支和合并使用菱形决策节点

### 连线规则
- 箭头方向表示数据流向
- 连线上标注数据格式/类型
- 使用不同颜色区分数据类型

### 标注规范
- 节点名称
- 数据格式（JSON/CSV等）
- 数据量级

## 示例场景

- 实时数据处理
- ETL 流程
- 报表生成流程
- 数据同步流程
