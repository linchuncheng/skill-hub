## CI/CD 集成：让发布流程自动化

这个 SKILL 最强大的地方，是可以**无缝集成到你的 CI/CD 流程中**。

### 典型场景：代码合并后自动升级数据库

```
开发者提交代码 → PR合并 → CI构建 → CD部署 → 自动执行数据库迁移
```

**以前你需要：**

```yaml
# .gitlab-ci.yml
stages:
  - deploy
  - migrate

deploy:
  stage: deploy
  script:
    - ./deploy.sh

migrate:
  stage: migrate
  script:
    - flyway migrate -url=$DB_URL -user=$DB_USER
    # 还要处理各种参数和错误...
```

**现在你只需要：**

```yaml
# .gitlab-ci.yml
stages:
  - deploy
  - migrate

deploy:
  stage: deploy
  script:
    - ./deploy.sh

migrate:
  stage: migrate
  script:
    # 通过 qodercli 调用 SKILL，一句话搞定
    - qodercli -p "/agent-dbmate 执行数据库迁移 --config=test"
```

### 实际工作流

**场景 1：开发环境自动迁移**

```bash
# 开发者合并代码后，CI 自动执行
qodercli -p "/agent-dbmate 升级DB"

# AI 自动完成：
# ✅ 检查环境
# ✅ 发现待迁移的模块
# ✅ 按顺序执行
# ✅ 输出结果
```

**场景 2：测试环境一键升级**

```bash
# 测试前，一键升级数据库到最新版本
qodercli -p "执行数据库迁移 --config=test"

# 如果需要回滚
qodercli -p "回滚最后一个迁移 --config=test"
```

**场景 3：生产环境可控发布**

```bash
# 先看状态
qodercli -p "查看迁移状态 --config=prod"

# 确认无误后执行
qodercli -p "执行数据库迁移 --config=prod"

# 有问题立即回滚
qodercli -p "回滚最后一个迁移 --config=prod"
```

### 为什么这样更好？

| 传统方式 | SKILL 集成 |

| 写几十行 Pipeline 脚本 | 一行命令 |
| 脚本出错要调试 | AI 自动诊断 |
| 换环境要改配置 | 一个参数搞定 |
| 日志分散，难排查 | AI 输出清晰报告 |
